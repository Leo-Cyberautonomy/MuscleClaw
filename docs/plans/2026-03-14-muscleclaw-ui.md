# MuscleClaw — UI/UX 设计

> 基于：`2026-03-05-muscleclaw-ideation.md` + `2026-03-14-muscleclaw-design.md`
> 平台：Web（桌面浏览器为主，全屏使用）
> 交互方式：语音（主要）、手势（辅助）、鼠标（回退）

## 设计理念

MuscleClaw 不是传统多页面 Web 应用。它是一个**单屏幕、模式驱动**的实时系统：

- 摄像头画面**永远**是主画面，占据视觉焦点
- 用户通过**语音和手势**操作，不需要看键盘或点击小按钮
- 界面元素**叠加**在摄像头画面上（AR 风格），而不是替代它
- 信息根据当前模式**渐进式展示**，默认最简洁

**设计参考**：Iron Man Jarvis HUD — 深色透明 + 发光线条 + 浮动信息卡 + 语音驱动

---

## 信息架构（模式架构）

MuscleClaw 没有传统的"页面导航"。取而代之的是**模式状态机**——用户始终停留在同一个屏幕上，通过语音/手势切换模式，每个模式决定了叠加层和侧边栏的内容。

```
Main Screen（摄像头永远在）
│
├── Mode: 待机 (idle)
│   ├── Overlay: 无（干净的摄像头画面）
│   ├── Sidebar: 收起
│   └── 等待: 语音唤醒 / 人体检测
│
├── Mode: 身体扫描 (body_scan)
│   ├── Overlay: AR 身体面板（6 部位毛玻璃卡片 + 引导线）
│   ├── Sidebar: 身体档案详情 + 训练日历
│   └── 触发: "帮我安排训练" / 人体检测后自动
│
├── Mode: 训练规划 (planning)
│   ├── Overlay: AR 身体面板（高亮推荐部位）
│   ├── Sidebar: 训练计划表格
│   └── 触发: AI 生成训练计划后自动切换
│
├── Mode: 体态评估 (posture)
│   ├── Overlay: 骨骼线 + 体态标注（偏差角度、不对称标记）
│   ├── Sidebar: 体态报告 + 推荐热身 + 前后对比
│   └── 触发: "体态评估" / AI 主动建议
│
├── Mode: 训练中 (training)
│   ├── Overlay: 骨骼线 + 关节角度 + Rep 计数器 + 杠铃轨迹
│   ├── Sidebar: 当前训练进度 + 力量曲线图
│   ├── 子状态: 做组中 / 组间休息
│   └── 触发: "开始训练" / 竖大拇指
│
├── Mode: 展示 (showcase)
│   ├── Overlay: 摆姿骨骼指引 + 截图框
│   ├── Sidebar: AI 生图结果（before/after）
│   └── 触发: "进入展示模式"
│
└── Global（任何模式下可触发）
    ├── 安全警报覆盖层 (Safety Alert)  — 最高优先级，全屏
    ├── 休息计时器 (Rest Timer) — 悬浮圆形
    ├── 实时字幕条 (Transcript Bar) — 底部
    └── 设置面板 (Settings) — 滑出面板
```

### 模式切换方式

| 切换方式 | 操作 | 示例 |
|----------|------|------|
| **语音** | 直接说目标模式/动作 | "帮我安排训练""体态评估""进入展示模式" |
| **手势** | 挥手 = 回到上一步/切换 | 训练中挥手 → 回到规划 |
| **AR 按钮** | 食指点击浮动按钮 | 点击"开始训练"按钮 |
| **自动** | 系统根据上下文自动切换 | 检测到人体 → 身体扫描；生成计划 → 规划模式 |
| **鼠标** | 点击侧边栏/按钮（回退方案） | 点击侧边栏切换标签 |

## 用户流程

### 流程 1：首次使用（Onboarding）

```
触发：首次打开应用

1. 用户看到：全屏摄像头请求权限弹窗（浏览器原生）
   → 如果拒绝：显示权限引导页（文字说明 + 重试按钮）

2. 用户允许摄像头 + 麦克风
   → 摄像头画面全屏显示（干净，无任何覆盖）

3. AI 语音："你好！我是 MuscleClaw，你的 AI 训练搭档。
   先帮你建个档案吧？说'好的'或者竖个大拇指就行。"
   → 底部字幕条显示 AI 说的话

4. 用户说"好的" / 竖大拇指
   → 侧边栏滑出，显示档案设置表单

5. AI 逐项语音引导：
   - "你主要练什么？力量训练还是健身？"
   - "卧推最大重量大概多少？"
   - "设一个紧急联系人号码吧，以防万一"
   - "喜欢什么风格？正经的、温柔的、还是毒舌的？"（演示三种声音）
   → 用户语音回答，AI 自动填入表单
   → 用户也可以直接在侧边栏表单里鼠标操作

6. AI："档案建好了！随时对我说'帮我安排训练'就可以开始。"
   → 侧边栏收起
   → 进入待机模式

完成：用户档案已创建，系统进入待机模式
异常：
- 摄像头拒绝 → 显示图文引导页，说明如何在浏览器设置中允许
- 麦克风拒绝 → 退化为文本输入模式，侧边栏底部显示输入框
```

### 流程 2：日常训练（核心主流程）

```
触发：用户走到摄像头前

1. 系统检测到人体
   → 底部短暂闪现提示："已检测到你"
   → 如果 > 3 天没训练，AI 主动打招呼："好久不见啊！今天练什么？"

2. 用户说"帮我安排今天的训练"
   → 切换到 body_scan 模式
   → AR 身体面板动画渐入（6 个卡片从中心向外展开）
   → 每个卡片显示：部位名称、代表力量、上次训练、恢复状态

3. AI 语音分析：
   "你的胸肌上次练是三天前，超量恢复完成了。
   背部是弱项，推荐今天练胸+背。"
   → body_scan 模式中，推荐部位的卡片边框高亮（蓝色脉冲）

4. AI 生成训练计划
   → 自动切换到 planning 模式
   → 侧边栏显示训练计划表格
   → AR 身体面板保留但淡化

5. 用户说"开始" / 竖大拇指 / 点击"开始训练"AR 按钮
   → 切换到 training 模式
   → AR 身体面板淡出
   → 训练 HUD 淡入（骨骼线 + Rep 计数器）
   → 侧边栏切换为训练进度视图

6. 训练中：每一组的循环 → 见流程 3

7. 所有动作完成
   → AI："今天的训练全部完成！干得不错！"
   → 侧边栏显示训练总结
   → 提议："要不要进展示模式？来秀一下肌肉？"

完成：训练数据自动保存，用户可选择展示模式或结束
```

### 流程 3：训练中一组动作

```
触发：训练模式下，当前动作/组开始

1. 侧边栏顶部显示：
   当前动作名 | 目标组数/当前组 | 建议重量
   例：卧推 | 第 2/4 组 | 105kg × 6

2. 用户开始做动作
   → CV 检测到动作类型匹配（卧推角度模式）
   → 骨骼线激活为亮色
   → Rep 计数器归零，显示在画面左上角

3. 每完成一个 Rep：
   → 计数器 +1，数字放大动画
   → AI 语音报数：
     - 正经模式："第三下"
     - 嘲讽模式："第三下！就这？还是热身吧？"
   → 如果 ROM 不够：画面闪红 + AI "嗯嗯不算！手不够直！"
   → 如果不对称：问题关节标红 + AI "左手低了，抬高一点"

4. 一组结束（用户放下重量 / 竖大拇指 / 语音"好了"）
   → AI 确认："这组 105kg 做了 6 个，对吗？"
   → 用户确认 / 纠正
   → 数据自动记录
   → 休息计时器出现（画面中央浮动圆形倒计时）

5. 休息倒计时中：
   → 圆形进度环倒计时（默认 120 秒）
   → 侧边栏显示上一组数据 + 力量曲线
   → AI 可能闲聊或给建议："下一组试试加 2.5kg？"

6. 倒计时结束
   → 计时器消失
   → AI："休息够了，来，下一组！" / 嘲讽："你是在休息还是在度假？开工！"
   → 回到步骤 2

完成：当前动作所有组完成 → 自动进入下一个动作
异常：
- 中途想换动作 → 语音"换一个" → AI 建议替代动作
- 想提前结束 → 语音"这个动作不做了" → 跳到下一动作
```

### 流程 4：安全检测

```
触发：CV SafetyMonitor 检测到危险信号

1. 全屏半透明红色覆盖层立即出现
   → 中央大字："⚠ 检测到异常"
   → 倒计时数字（10 秒）
   → 所有其他 UI 元素暂时隐藏

2. AI 语音（紧急语气，所有性格模式统一切换为严肃）：
   "检测到你可能卡住了！还好吗？
   说'取消'或'我没事'，否则 10 秒后拨打紧急联系人。"

3. 倒计时进行中，每秒更新数字

4a. 用户回应"我没事" / "取消" / "我逗你的"
    → 覆盖层消失（快速淡出）
    → AI："好吧好吧，吓我一跳！" / 嘲讽："你小子，差点让我打 120。"
    → 恢复之前的模式

4b. 10 秒无回应
    → 覆盖层变为："正在拨打紧急联系人..."
    → 跳转 tel: URI / 后端通知
    → AI："别担心，救援马上来"

异常：
- 紧急联系人未设置 → 覆盖层显示："请设置紧急联系人" + 输入框
- 误触频繁 → AI 建议调整灵敏度
```

### 流程 5：体态评估

```
触发：用户说"体态评估" / AI 建议"先做个体态评估？"

1. 切换到 posture 模式
   → 骨骼线亮起
   → 画面中央显示引导框（人形轮廓 + 站位指引）

2. AI 引导："请面向摄像头站直，双手自然下垂"
   → 画面显示对齐指引（中线 + 双脚位置标记）
   → 用户站好后 → 自动截图 + 分析
   → 画面闪一下表示拍照

3. AI："好，现在向左转 90 度"
   → 重复：引导 → 对齐 → 截图
   → 共四个角度：正面、左侧、背面、右侧

4. 分析完成
   → 骨骼线上标注问题区域（红色标记 + 偏差角度数字）
   → 侧边栏显示体态报告：
     - 每个检测项（骨盆前倾、肩部对称、脊柱、肱骨位置）
     - 状态：✓ 正常 / ⚠ 轻度 / ✗ 需关注
     - 推荐热身动作

5. AI 语音解读：
   "检测到右肩肱骨前移，今天做卧推时注意收紧肩胛骨。
   推荐先做这两个热身动作。"

6. 训练后再次评估（如果用户选择）
   → 侧边栏显示 before/after 对比（截图 + 分析数据并排）
   → AI："比刚才好多了！右肩前移改善了 3 度。"

完成：体态报告保存，可返回训练规划
```

### 流程 6：展示模式

```
触发：用户说"进入展示模式"

1. 切换到 showcase 模式
   → 训练 HUD 清除
   → 画面变得更"干净"（只保留骨骼线）

2. AI："来个蟹式（most muscular）吧！"
   → 画面显示目标姿势的骨骼参考线（半透明）

3. 用户摆姿势
   → 实时比对用户骨骼 vs 目标骨骼
   → AI 纠正："手臂再抬高一点""发力！""腰挺直"
   → 匹配度达标 → AI："好，保持！"
   → 自动截图（画面闪光 + 快门音效）

4. AI（嘲讽模式）："检测到你毫无训练痕迹，现在帮你上科技！"
   → 侧边栏显示"处理中..."加载动画
   → ImageGenAgent 用 Nano Banana 2 编辑截图（增强肌肉）

5. 生成完成
   → 侧边栏显示 before/after 对比
   → AI："看看这个科技效果！以后真练成这样，就不用我了。"

完成：生图结果可保存/分享
异常：
- 生图失败 → AI："科技注入失败，你的肌肉太顽固了。再来一次？"
```

## 屏幕区域设计

### 主布局

MuscleClaw 只有一个屏幕，分为以下区域：

```
┌──────────────────────────────────────────────────────┬──────────────┐
│                                                      │              │
│                                                      │              │
│                   摄像头主画面                          │    侧边栏     │
│              (100% - sidebar width)                   │   (360px)    │
│                                                      │              │
│    ┌─ Canvas 2D Layer (骨骼/角度/轨迹/AR按钮) ─┐      │              │
│    └──────────────────────────────────────────┘      │              │
│                                                      │              │
│    ┌─ HTML Float Layer (毛玻璃卡片/安全覆盖) ──┐      │              │
│    └──────────────────────────────────────────┘      │              │
│                                                      │              │
├──────────────────────────────────────────────────────┤              │
│                  实时字幕条 (48px)                     │              │
└──────────────────────────────────────────────────────┴──────────────┘
```

### 区域 1：摄像头主画面

**层级（从下到上）**：

| Z 层 | 内容 | 技术 |
|------|------|------|
| z-0 | `<video>` 摄像头实时画面 | HTML5 Video |
| z-1 | Canvas 2D — 骨骼线、角度弧线、轨迹、AR 按钮 | Canvas API |
| z-2 | HTML 浮动层 — 身体面板卡片、Rep 计数器、休息计时器 | HTML/CSS absolute positioned |
| z-3 | 安全警报覆盖层（最高优先级） | HTML/CSS fixed overlay |

### 区域 2：侧边栏 (360px)

**位置**：屏幕右侧固定
**高度**：全屏高度
**背景**：深色半透明（`rgba(10, 10, 15, 0.85)` + `backdrop-filter: blur(20px)`）
**边框**：左侧 1px `rgba(255,255,255,0.06)`

**侧边栏头部（固定）**：
- 当前模式标题（如"训练中 · 卧推"）
- 模式图标
- 收起按钮（←）

**侧边栏内容（随模式切换）**：

| 模式 | 侧边栏内容 |
|------|-----------|
| idle | 收起状态（只显示一个展开箭头 →） |
| body_scan | 身体档案卡片列表 + 训练日历热力图 |
| planning | 训练计划表格（动作/组数/重量） |
| posture | 体态报告 + 推荐热身列表 + 对比图 |
| training | 当前组进度 + 上一组数据 + 力量曲线小图 |
| showcase | 截图预览 + AI 生图结果 + before/after |

**侧边栏可收起/展开**：
- 默认：根据模式自动展开/收起
- 手动：点击边缘箭头 / 语音"收起侧边栏" / 挥手手势
- 收起时：主画面扩展为全屏宽度

### 区域 3：实时字幕条

**位置**：主画面底部
**高度**：48px
**样式**：半透明深色条（`rgba(0,0,0,0.6)` + `backdrop-filter: blur(8px)`）
**内容**：
- 左侧：用户说的话（白色，带渐入动画）
- 右侧：AI 说的话（品牌色/青色，带渐入动画）
- 字幕在 3 秒后自动淡出

## 核心 UI 组件设计

### 1. AR 身体面板（Body Panel）

**出现时机**：body_scan / planning 模式

**组成**：6 个浮动信息卡片 + 6 条引导线

**卡片布局策略**（避开身体）：

```
              [肩 Card]
               ╱
    [手臂 Card]─── ╲ ───[胸 Card]
                    │
              ┌─────┤
              │  人  │
              │  体  │
              │     │
    [背 Card]─┤     ├──[核心 Card]
              │     │
              └─────┘
                │
           [腿 Card]
```

**实际定位逻辑**：
- 计算对应身体部位的 landmark 中心点
- 卡片放在中心点的水平偏移方向（左侧部位卡片放左边，右侧放右边）
- 垂直位置与 landmark 大致对齐
- 保证卡片不超出屏幕边界（边缘碰撞约束）
- 引导线从 landmark 中心到卡片最近边缘，用贝塞尔曲线

**单个卡片设计**：

```
┌──────────────────────────┐
│  ● 胸部                   │ ← 部位名 + 状态圆点（绿/黄/红）
│  卧推 110kg               │ ← 代表力量
│  3 天前训练 · 已恢复       │ ← 最近训练 + 恢复状态
└──────────────────────────┘
```

- 尺寸：约 200×80px
- 背景：`backdrop-filter: blur(12px); background: rgba(15, 20, 30, 0.65)`
- 边框：`1px solid rgba(100, 200, 255, 0.15)`
- 圆角：12px
- 文字：白色，14px，行高 1.4
- 状态圆点颜色：
  - 🟢 绿 = 已恢复（可训练）
  - 🟡 黄 = 恢复中（可轻度训练）
  - 🔴 红 = 未恢复 / 有伤
- 被推荐训练的卡片：边框变为 `rgba(0, 180, 255, 0.6)` + 微弱脉冲动画

**引导线**：
- 颜色：`rgba(100, 200, 255, 0.3)`
- 宽度：1.5px
- 端点：带 4px 小圆点（landmark 端）
- 发光：下层画一条 4px 宽的 `rgba(100, 200, 255, 0.08)` 模糊线

**动画**：
- 面板出现：卡片从身体中心向外展开（stagger 动画，每张卡片延迟 80ms）
- 跟随移动：landmark 坐标经过 EMA（α=0.3）低通滤波后更新卡片位置
- 面板消失：卡片向身体中心收缩 + 淡出

### 2. 训练 HUD（Training Head-Up Display）

**出现时机**：training 模式

**组成**：

```
┌────────────────────────────────────────────────┐
│                                                │
│  ┌──────┐                                     │
│  │  5   │ ← Rep 计数器（左上角）               │
│  │ REPS │                                      │
│  └──────┘                                      │
│                                                │
│         ╱ 142° ╲  ← 关节角度标注（弧线+数字）  │
│        ╱        ╲                              │
│       ●──────────● ← 骨骼连线（发光效果）      │
│        ╲        ╱                              │
│         ╲      ╱                               │
│          ●    ●                                │
│          │    │    ← 杠铃轨迹（历史路径虚线）   │
│         ···  ···                               │
│                                                │
│              ┌──────────────────┐              │
│              │  ⏱ 1:45 REST     │ ← 休息计时器 │
│              └──────────────────┘              │
│                                                │
└────────────────────────────────────────────────┘
```

**Rep 计数器**：
- 位置：主画面左上角，距顶部 24px，距左 24px
- 数字：64px 粗体，白色，text-shadow 发光
- "REPS" 标签：14px，`rgba(255,255,255,0.5)`
- 背景：`rgba(0,0,0,0.4)` 圆角矩形
- 新 rep 动画：数字放大 120% → 弹回 100%，持续 300ms
- ROM 不达标时：数字变红 + 抖动

**骨骼连线**：
- 正常：`rgba(0, 255, 180, 0.7)` 2px 线 + 下层 `rgba(0, 255, 180, 0.15)` 6px 发光
- 问题关节：`rgba(255, 60, 60, 0.9)` + 闪烁（500ms 周期）
- 关键点圆点：4px，与线同色

**关节角度标注**：
- 跟踪的关节显示弧线 + 角度数值
- 弧线半径 30px，从一侧骨骼到另一侧
- 角度数字：12px，显示在弧线外侧
- 正常角度：白色
- 不达标：红色 + 闪烁

**杠铃轨迹**：
- 最近一个 rep 的轨迹：实线，`rgba(255, 200, 0, 0.7)` 2px
- 历史 rep 轨迹：虚线，`rgba(255, 200, 0, 0.2)` 1px
- 轨迹上的起点/终点：6px 圆点

### 3. 休息计时器（Rest Timer）

**出现时机**：训练模式 · 组间休息

**位置**：主画面正中偏下（垂直 60%，水平居中）

**设计**：

      │    ██████    │

        ╰───────────╯

## 各模式状态设计

### 待机模式 (idle)

| 状态 | 用户看到什么 | 用户能做什么 |
|------|-------------|-------------|
| **正常** | 干净的摄像头画面 + 底部淡色文字"说点什么，或者走到镜头前" | 语音、走到摄像头前 |
| **未连接** | 摄像头画面 + 右上角红色圆点"连接中..." | 等待自动重连 |
| **摄像头失败** | 深色背景 + 大图标 + "请允许摄像头访问" + 重试按钮 | 点击重试 |
| **麦克风失败** | 摄像头画面 + 底部显示文本输入框 | 打字输入 |

### 身体扫描模式 (body_scan)

| 状态 | 用户看到什么 | 用户能做什么 |
|------|-------------|-------------|
| **扫描中** | 骨骼线 + 扫描线动画（从上到下扫过身体） | 等待（2 秒左右） |
| **正常** | 6 个身体卡片 + 引导线 + 侧边栏档案 | 语音询问、手势操作、点击卡片 |
| **空状态**（首次） | 卡片显示"--"代替数据 + AI 语音引导填写 | 语音回答 AI 的问题 |
| **人离开画面** | 卡片淡出 + "已丢失追踪" | 回到画面内 |

### 训练模式 (training)

| 状态 | 用户看到什么 | 用户能做什么 |
|------|-------------|-------------|
| **做组中** | 骨骼线 + 角度 + Rep 计数器 + 侧边栏进度 | 做动作、语音交流 |
| **组间休息** | 休息计时器 + 上组数据 + 侧边栏曲线 | 等待 / 语音"跳过休息" |
| **动作纠正** | 问题关节闪红 + 角度标红 | 调整姿势 |
| **换动作过渡** | 短暂"下一个：深蹲"全屏提示(1.5s) | 准备下一个动作 |
| **训练完成** | 训练总结覆盖 + 侧边栏统计 | 语音选择展示模式 / 结束 |

### 体态评估模式 (posture)

| 状态 | 用户看到什么 | 用户能做什么 |
|------|-------------|-------------|
| **引导中** | 画面中央人形轮廓 + 站位标记 + 方向指引 | 按引导站好/转身 |
| **拍照中** | 画面闪白（快门效果） | 等待 |
| **分析中** | 骨骼线 + 旋转加载动画 | 等待 |
| **结果展示** | 骨骼线 + 问题标注 + 侧边栏报告 | 查看报告、语音询问 |
| **对比模式** | 画面分屏（before/after） + 侧边栏数据对比 | 查看 |

### 展示模式 (showcase)

| 状态 | 用户看到什么 | 用户能做什么 |
|------|-------------|-------------|
| **摆姿引导** | 目标姿势骨骼参考线（半透明蓝色） | 照着摆 |
| **截图** | 画面闪白 + 快门音效 | 保持 |
| **AI 生图中** | 侧边栏显示加载动画 + 截图预览 | 等待 |
| **结果展示** | 侧边栏 before/after 对比 | 重新拍 / 保存 |
| **生图失败** | 侧边栏错误提示 + 重试按钮 | 重试 |

## 视觉设计规范

### 色彩系统

| 用途 | 色值 | 说明 |
|------|------|------|
| **主背景** | 摄像头画面 | 永远是主视觉 |
| **面板背景** | `rgba(10, 15, 25, 0.65)` | 深色半透明 |
| **侧边栏背景** | `rgba(10, 10, 15, 0.85)` | 更深更不透明 |
| **品牌色/科技蓝** | `rgba(0, 180, 255, 1.0)` | 推荐高亮、引导线、品牌元素 |
| **正常骨骼** | `rgba(0, 255, 180, 0.7)` | 绿色系，表示正常 |

## 响应式策略

MuscleClaw 设计为**桌面全屏使用**（健身房的大屏幕或笔记本），但需要考虑不同屏幕尺寸：

| 屏幕宽度 | 布局调整 |
|----------|---------|
| ≥ 1280px | 标准布局：主画面 + 360px 侧边栏 |
| 1024-1279px | 侧边栏缩为 300px，卡片文字缩小 |
| 768-1023px | 侧边栏默认收起，需手动展开，展开时覆盖主画面 |
| < 768px | 不支持提示："请在更大的屏幕上使用 MuscleClaw" |

**不做移动端适配**（理由：健身场景需要大屏幕 + 足够远的距离才能看到全身，手机屏幕太小且距离太近）

## 数据需求汇总

此列表是 API 设计阶段的直接输入。

### 读取需求

| 页面/模式 | 需要的数据 | 数据结构 |
|-----------|-----------|---------|
| body_scan | 六大部位身体档案 | `{part: string, max_weight: number, exercise: string, last_trained: date, recovery_status: enum}[]` |
| body_scan | 训练日历数据 | `{date: string, trained_parts: string[]}[]`（最近 90 天） |
| planning | 训练计划 | `{exercises: {name: string, sets: number, reps: number, weight: number}[]}` |
| training | 当前动作的历史力量数据 | `{date: string, sets: {reps: number, weight: number, rpe: number}[]}[]` |
| training | 动作定义（角度阈值等） | `{name: string, tracking_joints: string[], rom_threshold: number, ...}` |
| posture | 体态评估结果 | `{items: {name: string, status: enum, angle: number, note: string}[]}` |
| posture | 历史体态对比 | `{before: PostureResult, after: PostureResult}` |
| showcase | AI 生图结果 | `{original_url: string, enhanced_url: string}` |
| settings | 用户偏好 | `{personality: enum, language: string, emergency_contact: string, rest_timer: number, volume: number, safety_sensitivity: enum}` |

### 写入需求

| 操作 | 触发方式 | 数据 |
|------|---------|------|
| 创建用户档案 | onboarding 完成 | 完整 body_profile + preferences |

| 记录训练组数据 | 每组结束 | `{exercise, reps, weight, rpe, rom_avg}` |

| 更新偏好设置 | 设置面板修改 | 变更的字段 |
